# แผนพอร์ต GAS: COSINE closed-form + arc-length inversion → reference/gsheet/

## Context

`session_logs/investigate_gsheet_port_scope.md` (commit `9d5eeea`) สืบสวนสโคปงานนี้ไว้แล้ว
สรุปว่า Python core engine (`src/smt/alignment.py`) มีสองส่วนที่ oracle `.gs` เดิมไม่มี:
COSINE closed-form (Civil 3D Sine Half-Wave, EXT-003) + arc-length inversion (bisection)
และ builder-side turning-angle fix ที่ตามมา ส่วนนี้ถูกพอร์ตไป VBA แล้วสำเร็จ (Phase 4,
commit `e285fd5`) แผนนี้พอร์ตส่วนเดียวกันไปเป็น Google Apps Script ตาม pattern
`reference/vba/` ที่ตกลงกันไว้: **ไม่แตะ `reference/*.gs` เดิมแม้แต่บรรทัดเดียว** (frozen
oracle ผ่าน AllTests 45/45) — งานทั้งหมดอยู่ในไฟล์ใหม่ที่ `reference/gsheet/` prefix `GS_`

ขอบเขตที่ตัดสินใจแล้ว (เหมือน VBA Phase 4 เป๊ะ):
- รวม: COSINE closed-form + arc-length inversion เท่านั้น
- ไม่รวม: EXT-002 (`fit_radius`/Nelder-Mead), LandXML export, `GS_Vertical`/`GS_Crossfall`/ฯลฯ

การตัดสินใจเพิ่มเติมจากคำถามที่ผู้ใช้ตอบแล้ว:
1. **`GS_AlignmentBuilder.gs` ตั้งฐานจาก `reference/AlignmentBuilderV2.gs`** (ซึ่งมี EXT-001
   angle-point อยู่แล้ว) — `git mv` เข้า `reference/gsheet/` ก่อน แล้วแพตช์ EXT-003
   turning-angle ทับ ได้ feature parity เต็มกับ Python builder ปัจจุบัน (มีทั้ง EXT-001+EXT-003
   ในไฟล์เดียว) และแก้ปัญหาชื่อ/ตำแหน่งไฟล์ไม่สอดคล้องที่รายงานสืบสวนข้อ 3 ทิ้งไว้เป็นคำถามเปิด

### ขั้นตอน

1. Copy ทั้งไฟล์ 268 บรรทัดจาก `Alignment.gs` → `GS_Alignment.gs` เปลี่ยน
   `var Alignment = (function () {` (บรรทัด 24) และจุดอ้างอิงชื่อ module ท้ายไฟล์เป็น

2. เพิ่ม private helper functions ใหม่ (mirror ชื่อ Python, ใช้ camelCase + trailing
   underscore ตาม convention ของไฟล์เดิมเช่น `shapeIntegral_`):

     Simpson quadrature; ต้องอ่านค่าคงที่ `SPIRAL_STEPS`/`n` จริงจาก branch spiral เดิมของ
     `pointOnElement` (บรรทัด 114-133) ตอน execute — **ห้ามเดา/hardcode 48**
   - `cosineArcLengthTable_(length, rAbs)` — mirror `_cosine_arc_length_table` (Python
     `:170-181`) ใช้ **`Map`** แทน `lru_cache` คีย์ `String(length) + '|' + String(rAbs)`
     (ไม่ round — ตรงตาม float key ของ Python tuple `(length, r_abs)`)
   - `cosineSolveA_(d, bigX, r, length)` — mirror `_cosine_solve_a` (Python `:184-212`)
     bracket จาก cached table แล้ว bisect 50 รอบ (ต่างจาก VBA ที่ไม่ cache — GAS ใช้ cache
     ตามที่ investigation ยืนยันว่า `Map` ใช้งานได้จริงใน V8 runtime)
   - `calcSineHalfwaveTangentLength(length, r)` — **ไม่มี underscore** (public) mirror
     `calculate_sine_halfwave_tangent_length` (Python `:244-261`) เพราะจะ expose เป็น UDF
     ด้วย; ค่าคงที่ `_SINE_HALFWAVE_C = 0.0226689447`
   - `sineHalfwavePoint_(d, bigX, r, length)` — mirror `_sine_halfwave_point` (Python
     `:215-241`) คืน `{x, y, theta}`

3. เพิ่ม branch ใหม่ใน `pointOnElement` ก่อน spiral-Simpson `else` (mirror Python
   `:378-401` และ VBA `SMT_Alignment.bas:250-279`):

   ```

   SPIN  (kIn===0): r = radiusFromCurvature(el.kOut)

                    {x,y,theta} = sineHalfwavePoint_(d, bigX, r, length)
   ```

   แปลงกลับเป็น N/E ด้วยสูตร rotation **ที่มีอยู่แล้วในไฟล์** (copy ตรงจาก branch อื่นของ

   ```js
   /** @customfunction */
   ```

5. `shapeIntegral_` เดิมคงไว้ทั้งหมดไม่แตะ (fallback สำหรับ compound COSINE)

## 4) Node smoke-test (`reference/gsheet/smoke_test.js`)

- `require('./GS_Alignment.gs')` ตรงๆ แบบ CommonJS (oracle `.gs` ทำได้แล้วตามที่
  investigation ยืนยัน — ต้องทดสอบจริงตอน execute ว่า Node `require` ไฟล์นามสกุล `.gs`
  ด้วย path เต็มได้เลยหรือไม่)
- สร้าง element SPIN สังเคราะห์ (`n=0, e=0, azimuth=0`) เรียก `pointOnElement(el, length)`
  ที่ `d=length` → `x=result.n, y=result.e` (เพราะ `az=0` ทำให้ rotation เป็น identity),
  `theta_deg = radToDeg(result.azimuth)`
- เทียบกับ 3 จุด ground truth ด้วย tolerance `1e-9` (เท่า pytest `abs_tol`)
- ทดสอบ SPOUT ด้วย synthetic element กลับด้าน เทียบ symmetry
- ทดสอบ cache ด้วย `cosineCacheSize_()`
- exit code `0` ถ้าผ่านหมด, non-zero + print ค่าที่ผิดถ้า assertion fail
- รันด้วย `node reference/gsheet/smoke_test.js`
